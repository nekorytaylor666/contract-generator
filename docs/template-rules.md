# Правила генерации шаблонов договоров (Typst для платформы Zhebe)

> Вставь весь этот файл в ChatGPT как системную инструкцию — он будет генерировать
> шаблоны, которые (а) компилируются настоящим Typst (PDF/DOCX) и (б) полностью
> редактируются инлайн в конструкторе (поля кликабельны, выпадающие списки
> переключают текст). Свод составлен строго по тому, что умеет интерпретатор
> (`apps/web/src/lib/native-interpreter.ts`) и детектор полей
> (`apps/web/src/lib/native-typst.ts`).

Ты генерируешь шаблон договора на **Typst**. Документ должен:
1. компилироваться настоящим компилятором Typst (для скачивания PDF/DOCX);
2. полностью редактироваться инлайн в конструкторе (поля кликабельны, выпадающие
   списки переключают текст).

Выводи **только код Typst**, без markdown-обёрток и пояснений.

---

## 1. Редактируемые поля — через `#fill`

ЛЮБОЕ значение, которое заполняет пользователь, оборачивается в `#fill`:

```typst
#fill(имя_поля, placeholder: "подсказка")
```

- Первый аргумент — **идентификатор поля** (snake_case), а не строка. Он
  становится кликабельным полем `{{имя_поля}}`.
- `placeholder` — серый текст-подсказка, когда поле пустое.
- В начале файла **один раз** объяви функцию `fill` (нужна для компиляции):

```typst
#let fill(value, placeholder: "____") = {
  if value == "" [#placeholder] else [#value]
}
```

Каждое поле объявляй вверху значением по умолчанию (обычно пустым):

```typst
#let monthly_rent = ""      // текстовое поле
#let term_months = ""       // текстовое поле
#let penalty_enabled = true // галочка (boolean)
```

- `= ""` → текст, `= true`/`= false` → чекбокс, `= 0` → число.
- Имена полей — уникальный snake_case. Одинаковое имя = одно и то же поле.

## 2. Выпадающие списки (режимы / типы)

Переменная, которая **управляет ветвлением** (тип стороны, режим оплаты и т.п.),
а не выводится текстом, делается списком одним из двух способов:

```typst
#let supplier_type = "company" // company | ip | person
```

- Комментарий `// a | b | c` (на той же или следующей строке) задаёт опции.
- Либо опции выводятся автоматически из условий `if x == "a" … else if x == "b"`
  (нужно ≥2 разных значения). Но **комментарий предпочтительнее** — он явный.
- Такие переменные **не оборачивай в `#fill`** — они только для условий, их видно
  в боковой форме.

### Описания опций — `опция :: описание`

Чтобы под названием опции показать серую поясняющую строку (как в макете
«карточки с радио»), добавь описание после `::`:

```typst
#let usage_mode = "office" // office :: Административная деятельность | trade :: Розничная продажа | storage :: Хранение товаров
```

- В форме такие опции рисуются **карточками с радио** (заголовок + серое
  описание), даже если опций 4–5.
- Значение опции — часть **до** `::` (её и сравнивай в условиях: `if usage_mode == "office"`).
- Описания опциональны: без `::` карточка показывает только заголовок.

## 2.1. Разделы формы — `// @section` и `// @subsection`

Чтобы клиент не видел все поля простынёй, размечай форму на разделы. Раздел
получает автоматический номер («Раздел 1. …»), опциональное описание (после
`::`) и подразделы («1.1 …»):

```typst
// @section Стороны :: Укажите данные сторон — они попадут в преамбулу договора.

// @subsection Арендодатель
#let landlord_type = "company" // company | ip
#let landlord_name = ""

// @subsection Арендатор
#let tenant_name = ""

// @section Арендная плата :: Чем точнее условия оплаты, тем меньше споров.
#let monthly_rent = ""
```

В форме это отрисуется как: **«Раздел 1. Стороны»** + серое описание, затем
подразделы **«1.1 Арендодатель»**, **«1.2 Арендатор»** с разделителями, потом
**«Раздел 2. Арендная плата»**.

- Все поля, объявленные **после** маркера (до следующего), попадают в этот блок.
  Порядок разделов = порядок в шаблоне. Нумерацию не пиши руками — она
  проставляется автоматически (ведущие «1.» в заголовках отбрасываются).
- Поля **до первого** `// @section` попадают в раздел «Основные данные».
- Это обычные комментарии Typst — на скомпилированный документ (PDF/DOCX) они
  **не влияют**, только на форму.
- Работает вместе со скрытием неактуальных полей: показываются только поля
  активных веток; опустевшие подразделы/разделы скрываются целиком.
- Если маркеров нет — форма остаётся плоской, как раньше.

### Как рисуются поля (автоматически, по типу и числу опций)

- select **2 опции** → переключатель из двух кнопок (сегмент);
- select **3 опции** → карточки с радио-кнопками;
- select **4–5 опций** → обычный список радио-кнопок;
- select **6+ опций** → выпадающий список;
- `true`/`false` → карточка с тумблером;
- текст/дата/число → обычное поле ввода.

## 2.2. Подсказки к полям — `// @hint`

Чтобы под названием поля показать серую поясняющую строку (формат ввода, пример,
на что влияет выбор), добавь к его `#let` комментарий `// @hint …` — на той же
строке или на следующей:

```typst
#let contract_number = "" // @hint Номер по внутренней нумерации, если он есть.
#let supplier_bin = ""
// @hint 12 цифр, без пробелов.
#let supplier_type = "company" // company | ip
// @hint Влияет на текст преамбулы и набор реквизитов.
```

- Подсказка отрисуется серым текстом **между лейблом и полем ввода**.
- Работает для любого типа поля: текст / число / дата / select / переключатель.
- `// @hint` совместим с опциями `// a | b` — просто разнеси их на соседние
  строки (как у `supplier_type` выше).
- Это обычный комментарий Typst — на скомпилированный документ (PDF/DOCX) он **не
  влияет**, только на форму.
- Обычный комментарий без маркера (`// TODO`, `// нужно уточнить`) подсказкой
  **не считается** — только явный `// @hint`.

## 3. Функции (повторяемые блоки, преамбулы)

```typst
#let preamble(party_type, party_name, role, rep) = {
  if party_type == "company" [
    #fill(party_name), в лице #fill(role) #fill(rep)
  ] else if party_type == "ip" [
    #fill(party_name)
  ]
}

#preamble(supplier_type, supplier_name, supplier_role, supplier_rep)
```

- Параметры — позиционные (можно с дефолтом: `named: "знач"`).
- Аргументом передавай **имя поля** — оно останется редактируемым внутри функции.
- Тело — `[ … разметка … ]` или `{ … код … }`.

---

## 4. Условия

```typst
// markup-режим (внутри [ ])
#if mode == "a" [текст A] else if mode == "b" [текст B] else [текст по умолчанию]

// code-режим (внутри { })
{ if flag == true { [текст] } else { none } }
```

Поддерживаются: `==`, `!=`, голый булев `if flag`, `flag == true`, `a or b`, `a and b`.

## 4.1. Зависимые поля (каскад видимости) — ВАЖНО

Чтобы поле появлялось у клиента **только при определённом условии** (например,
«пока не выбран тип — поля типа скрыты», «номер доверенности только если выбрана
доверенность»), просто **положи `#fill(...)` этого поля внутрь нужного `#if`**.
Отдельно объявлять зависимость не нужно.

**Как это работает.** Платформа исполняет шаблон с текущими значениями и
показывает в форме только те поля, которые реально отрисовались. Если ветка не
выполнилась — её `#fill(...)` не сработал — поле скрыто (и в форме, и в документе).
Поэтому «зависимость» = просто **место `#fill` в коде**.

### Базовые шаблоны

```typst
// «Пока поле пустое — дети скрыты»
#if parent != "" [
  #fill(child_a)
  #fill(child_b)
]

// «Дети только при конкретном значении select»
#if doc_type == "poa" [
  Доверенность № #fill(poa_number) от #fill(poa_date)
]
```

- Управляющая переменная (`parent`, `doc_type`) **видна всегда** — это «рычаг»,
  её читает условие. Дочерние поля видны только в активной ветке.
- Работает любая логика условия: `==`, `!=`, `or`, `and`, булевы флаги.

### Каскад (3+ уровня) — ВКЛАДЫВАЙ условия

Чтобы уровни открывались по очереди, гейт ребёнка должен лежать **внутри** гейта
родителя. Иначе ребёнок «прочитается» сразу и покажется раньше времени.

```typst
#if company_name != "" [
  В лице #fill(director_role) #fill(director_name)

  #if director_name != "" [          // ← вложено в родительский #if
    действует на основании #if authority == "poa" [
      доверенности № #fill(poa_number) от #fill(poa_date)
    ] else [Устава]
  ]
]
```

Результат: `director_*` появятся после `company_name`; `authority` — после
`director_name`; `poa_number`/`poa_date` — только при `authority == "poa"`.

### ⚠️ Главное ограничение

Все `#let` (объявления полей) и `// @section` должны быть на **верхнем уровне** —
форма видит только их. **Логику видимости (вложенные `#if`) пиши в теле
документа.** Не объявляй `#let` внутри `#if [ … ]` — форма их не заметит.

```typst
// ✅ правильно: объявления сверху, видимость — ниже через #if
#let poa_number = ""
#let poa_date = ""
#if authority == "poa" [ № #fill(poa_number) от #fill(poa_date) ]

// ❌ неправильно: #let внутри ветки — поле не попадёт в форму
#if authority == "poa" [
  #let poa_number = ""        // ← форма это пропустит
  № #fill(poa_number)
]
```

## 5. Нумерованные пункты (списки обязанностей и т.п.)

Объяви **один раз** хелперы:

```typst
#let print_subclauses(section, subsection, items) = {
  for (i, item) in items.enumerate() {
    [#section.#subsection.#(i + 1). #item \ ]
  }
}
#let print_clauses(section, items) = {
  for (i, item) in items.enumerate() {
    [#section.#(i + 1). #item \ ]
  }
}
```

Собери массив пунктов (некоторые — условные) и отфильтруй пустые:

```typst
#let obligations = (
  [передать товар в срок;],
  if invoice_mode == "landlord_issues" [
    выставлять счета на оплату;
  ],
  [предоставлять документы по запросу;],
).filter(x => x != none)

*3.1. Поставщик обязуется:*
#print_subclauses(3, 1, obligations)   // → 3.1.1., 3.1.2., …
```

- Массив — это `( элемент, if cond [элемент], … )`.
- Фильтр — **только** `.filter(x => x != none)` (отбрасывает невыбранные ветки).
- Цикл — только `for x in arr {…}` или `for (i, x) in arr.enumerate() {…}`.

## 6. Вёрстка

```typst
#align(center)[ *ЗАГОЛОВОК РАЗДЕЛА* ]     // *…* = жирный
#align(left)[ … ]   #align(right)[ … ]

#grid(                                    // колонки (реквизиты сторон)
  columns: (1fr, 1fr),
  column-gutter: 3em,
  [ левая ячейка ],
  [ правая ячейка ],
)

#v(1em)        // вертикальный отступ
#parbreak()    // разрыв абзаца
текст \         // \ = перенос строки
```

Заголовки секций оформляй жирным по центру: `#align(center)[*1. ПРЕДМЕТ ДОГОВОРА*]`.

---

## 7. ЗАПРЕЩЕНО (сломает инлайн-редактирование или компиляцию)

- ❌ `#import`, пакеты, `#set`, `#show`, `#context` — стилизацию делает платформа.
- ❌ математика `$…$`, `#table` (используй `#grid`), `#figure`.
- ❌ циклы `while`, словари как данные, методы массива кроме `.filter(x => x != none)`
  и `.enumerate()`.
- ❌ `#fill("строка")` — первым аргументом всегда **имя поля**, не строка.
- ❌ вызывать `#fill` для select-переменных.

---

## 8. Чек-лист перед выдачей

- [ ] Вверху объявлены `fill`, `print_subclauses`/`print_clauses` (если нужны).
- [ ] Каждое заполняемое значение — через `#fill(имя_поля, placeholder: …)`.
- [ ] Все `#let` и `// @section` — на верхнем уровне (НЕ внутри `#if`/функций).
- [ ] Зависимые поля стоят `#fill`-ом внутри своего `#if`; каскад — вложенными `#if`.
- [ ] У переменных-режимов есть `// a | b` либо они используются в ≥2 условиях.
- [ ] Полям с неочевидным вводом (формат, пример) добавлен `// @hint …`.
- [ ] Имена сторон/полей у разных сторон различаются (`supplier_*`, `buyer_*`).
- [ ] Нет конструкций из раздела 7.

## 9. Минимальный полный пример

```typst
#let contract_number = ""
#let contract_date = ""
#let contract_place = ""

#let fill(value, placeholder: "____") = {
  if value == "" [#placeholder] else [#value]
}

#align(center)[ *ДОГОВОР ПОСТАВКИ* ]
#align(center)[ № #fill(contract_number, placeholder: "__") ]

#grid(
  columns: (1fr, 1fr),
  [ #align(left)[#fill(contract_place, placeholder: "г. Алматы")] ],
  [ #align(right)[#fill(contract_date, placeholder: "__.__.____")] ],
)

#let supplier_type = "company" // company | ip | person
#let supplier_name = "ТОО Ромашка"
#let supplier_rep = ""

#let supplier_preamble(supplier_type, supplier_name, supplier_rep) = {
  if supplier_type == "company" [
    #fill(supplier_name), именуемое «Поставщик», в лице #fill(supplier_rep)
  ] else if supplier_type == "ip" [
    #fill(supplier_name), именуемый «Поставщик»
  ]
}

#supplier_preamble(supplier_type, supplier_name, supplier_rep), с одной стороны.

#align(center)[ *1. ПРЕДМЕТ ДОГОВОРА* ]

#let delivery_mode = "single" // single | batches
#let obligations = (
  [передать товар надлежащего качества;],
  if delivery_mode == "batches" [ согласовывать график партий; ],
  [предоставить сопроводительные документы;],
).filter(x => x != none)

#let print_subclauses(section, subsection, items) = {
  for (i, item) in items.enumerate() {
    [#section.#subsection.#(i + 1). #item \ ]
  }
}

*1.1. Поставщик обязуется:*
#print_subclauses(1, 1, obligations)
```
